AWS · Terraform · Case Study

WordPress on AWS —
High-Availability Architecture

A production-grade WordPress deployment built entirely with Terraform — dual VPC isolation, auto-scaling EC2, shared EFS storage, and Multi-AZ RDS. Deployed end-to-end with a single terraform apply.

Region: us-west-2 Compute: t4g.small Graviton2 Database: RDS MySQL 8.0 Multi-AZ IaC: Terraform ≥ 1.7
01 — Overview

What Was Built

A fully automated, cloud-native WordPress stack split across two dedicated VPCs — one for the application tier, one for the database — connected via VPC Peering. Strong network isolation at the infrastructure level, not just at the firewall.

Everything from credentials to AMI selection is handled by Terraform at deploy time. No manual steps, no hard-coded passwords, no SSH keys.

02 — Components

What Each Layer Does

ALB

Internet-facing, across two AZs. Cookie-based stickiness for WordPress admin sessions. Health checks via /health.

EC2 ASG

t4g.small Graviton2 in private subnets. Min 1, desired 2, max 4. Nginx + PHP-FPM 8.1. No public IPs.

EFS

Shared /var/www/html mounted across all instances. Encrypted, IAM-authenticated, TLS in transit.

RDS MySQL

MySQL 8.0, Multi-AZ, in isolated VPC2. Reachable only from EC2 via VPC Peering on port 3306.

03 — Security

Defence-in-Depth

Security is layered from the network edge down to IAM — each layer independently restrictive so a misconfiguration at one layer doesn't expose everything below it.

  • No SSH. No Bastion.All access via SSM Session Manager. Zero open port 22 anywhere.
  • IMDSv2 Enforcedhttp_tokens = required, hop limit 1. Blocks SSRF attacks from instance metadata.
  • Least-Privilege IAMEC2 role scoped to a single Secret ARN and specific EFS filesystem ID only.
  • Zero Hard-Coded CredentialsDB credentials auto-generated at deploy, stored only in Secrets Manager.
  • All Storage EncryptedEBS, EFS, and RDS — AES-256. No exceptions.
  • Network ACLsThree NACLs (public, private app, private data) on top of security groups.
  • SG-to-SG RulesALB → EC2 and EC2 → EFS use security group references, not CIDRs.
  • Nginx Path BlockingRequests to .env, .git, .htaccess return 403 by default.

05 — Known Limitations

Honest About the Gaps

Trade-offs documented during the assessment — not discovered after.

Critical

No HTTPS

All traffic over HTTP. Needs ACM certificate, port 443 listener, and HTTP→HTTPS redirect on the ALB.

High

Single NAT Gateway

NAT in AZ-A only. AZ-A outage breaks outbound internet for AZ-B instances — SSM and Secrets Manager fail.

High

No RDS Backups

retention = 0, no final snapshot. Fine for dev — unacceptable for production data.

Medium

No WAF / CloudFront

No SQLi, XSS, or DDoS protection at the edge. Static assets served directly from EC2 via ALB.
